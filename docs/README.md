# Documentation Index

このディレクトリでは、オンボーディングから運用、アーキテクチャの意思決定履歴までプロジェクトの知識をカテゴリごとに整理しています。目的に合わせて以下のセクションを参照してください。

- **背骨（ハッピーパス）リファクタの入口**: `product/refactoring-backbone-happy-path-issues.md`
- **ドキュメント管理（参照ログ/削除）**: `operations/document-management.md`

- `getting-started/`: 開発を始めるための手順やチェックリスト
- `operations/`: デプロイやテスト、トラブルシューティングの実務ガイド
- `architecture/`: 現行アーキテクチャと設計判断の背景
- `product/`: プロダクトバックログや不具合修正の履歴
- `references/`: 技術スタックやAPI仕様などのリファレンス
- `ai-models/`: 役割別AIエージェントのふるまい定義

各カテゴリ内のREADMEや冒頭セクションを頼りに、目的に合ったドキュメントを探してください。
