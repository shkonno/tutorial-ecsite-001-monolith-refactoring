# 作業時間計測ルール（Work Time Tracking）

このドキュメントは、リファクタリングを進める上での第一指標（作業時間）を **ブレなく** 記録するための最小ルールです。  
計画の入口: `docs/refacroting/MASTER-PLAN.md`

---

## 1. 計測の目的

- ユースケース単位の作業時間が **跳ね上がっていないか** を検知する
- 「実装が難しい」ではなく「問題が難しい」箇所を特定し、次の分割に繋げる

---

## 2. 計測単位

- **ユースケース（UC）**: 評価可能な一連の作業の流れ（〜2日）
- **イシュー（Issue）**: UCを構成する要素（〜2時間）
- **タスク（Task）**: 具体作業（〜30分）

---

## 3. 記録対象（カテゴリ）

原則は **アクティブ作業時間** を第一指標とし、ノイズは分離して別記録します。

- **Active**: 実装/テスト/リファクタ/ドキュメント更新など、成果物に直結する作業
- **Wait**: CI待ち、ビルド待ち、環境起動待ち、E2E待ち
- **Research**: 調査/学習/読み物（コード/ドキュメント/外部情報）
- **Interrupt**: 割り込み対応（別件レビュー、障害対応など）

---

## 4. ルール（最小）

- 追加要件が入ったら **同一UCに混ぜない**（別UCとして切り出す）
- 待ち/調査/割り込みはActiveに混ぜない（比較可能性を守る）
- 単発の数値ではなく **直近N本の平均** など推移で見る

---

## 5. 記録フォーマット（テンプレ）

以下を `docs/operations/work-time-tracking.md` に追記するか、別ファイルでログとして運用します（どちらでも可）。

### レコード（1UC=1レコード目安）

|---|---|---:|---:|---:|---:|---:|---|---|
| YYYY-MM-DD | UC1 | I1-2 |  |  |  |  |  |  |
